# AGENTS.md

このリポジトリは、Codex向けの実用Agent Skillsを集めて育てるためのリポジトリです。

Codexは、このリポジトリで作業するとき、以下の方針に従ってください。

## 基本方針

- 回答、コメント、ドキュメントは原則として日本語で書く。
- 専門用語を使う場合は、短く補足説明を入れる。
- 1つのSkillは、1つの明確な開発作業に集中させる。
- Skillは、実際の開発作業で繰り返し使えるものを優先する。
- 既存の公開Skillを参考にする場合は、本文を安易にコピーせず、ライセンスと出典を確認する。
- APIキー、トークン、パスワードなどの秘密情報をコードやドキュメントに直接書かない。

## Skill作成ルール

- 新しいSkillは `skills/<skill-name>/SKILL.md` に作成する。
- `name` は小文字英数字とハイフンのみを使う。
- `description` は「いつ使うか」が分かる文にする。
- Skill本文は「目的」「使う場面」「使わない場面」「確認する入力」「手順」「安全上の注意」「出力形式」を基本構成にする。
- 手順は番号付きリストで書く。
- 曖昧な表現を避け、Codexがそのまま作業できる粒度で書く。
- 外部サービス連携、API連携、MCP連携、SaaS連携を前提にしたSkillは、初期構成には追加しない。

## scriptsを追加する場合

- 今回の初期作成では `scripts/` を追加しない。
- 将来 `scripts/` を追加する場合は、目的、危険性、実行条件を `SKILL.md` に明記する。
- 破壊的な操作を行うスクリプトは追加しない。
- ファイル削除、外部送信、認証情報の読み取りを行う場合は、必ず明示する。
- 実行前にユーザー確認が必要な操作は、その旨を手順に書く。
- 依存関係を増やす場合は、理由を説明する。

## 参考元・ライセンス

- 参考元がある場合は、`NOTICE.md` または該当Skill内に出典を書く。
- 他のリポジトリや公開Skillを参考にした場合は、ライセンスが明示されているか確認する。
- コピーではなく、自分の言葉で書き直す。
- コピー・改変が必要な場合は、元ライセンスに従う。
- ライセンス不明のものは取り込まない。

## README更新ルール

Skillを追加・変更した場合は、必要に応じて `README.md` のSkill一覧も更新してください。

READMEでは、各Skillについて次の情報を簡潔に書いてください。

- Skill名
- 用途
- 想定する利用場面
- 注意点

## 品質チェック

Skillを追加・更新したら、次を確認してください。

- `SKILL.md` が存在する
- frontmatterに `name` と `description` がある
- `name` がkebab-caseである
- `description` が具体的である
- 使う場面と使わない場面が明確である
- 手順が番号付きで書かれている
- 出力形式が明確である
- 安全上の注意が必要なSkillでは明記されている
- 参考元がある場合はライセンスと出典が記録されている
