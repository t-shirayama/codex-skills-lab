# Skill Name

## 目的

このSkillが解決する作業を1つに絞って書く。

## 使う場面

- このSkillを使う具体的な依頼や状況を書く。
- トリガー語や典型的な入力例を書く。

## 使わない場面

- 対象外の作業を書く。
- 外部サービス連携や別Skillに任せる作業があれば書く。

## 確認する入力

- 必要なファイル、差分、ログ、仕様、制約を書く。
- 不足している場合に何を確認するかを書く。

## 手順

1. 入力と依頼内容を確認する。
2. 対象範囲と制約を整理する。
3. 必要な調査や確認を行う。
4. 作業結果をまとめる。
5. 安全上の注意、未確認事項、次の行動を明示する。

## 安全上の注意

- 破壊的変更、依存関係追加、仕様変更、テスト不足、秘密情報の扱いに注意する。
- 必要に応じて、実行前にユーザー確認が必要な操作を書く。

## 出力形式

- 作業結果をどの形式で返すかを書く。
- 例: 箇条書き、レビューコメント、修正方針、チェックリスト、Markdown本文。
- 未確認事項や残るリスクを書ける欄を用意する。
