# Skill Review Checklist

Skillを追加・更新するときは、次の項目を確認します。

## 必須項目

- [ ] `SKILL.md` が存在する
- [ ] YAML frontmatterがある
- [ ] frontmatterに `name` がある
- [ ] frontmatterに `description` がある
- [ ] `name` がkebab-caseである
- [ ] `description` が具体的で、いつ使うSkillか分かる
- [ ] `description` に日本語・英語の代表的なトリガー語が含まれている
- [ ] 「目的」が明確である
- [ ] 「使う場面」が明確である
- [ ] 「使わない場面」が明確である
- [ ] 似たSkillとの使い分けが「使わない場面」に書かれている
- [ ] 「確認する入力」が書かれている
- [ ] 「手順」が番号付きリストで書かれている
- [ ] 「安全上の注意」がある
- [ ] 「出力形式」が明確である
- [ ] 「出力形式」に未確認事項、リスク、注意点のいずれかを書ける欄がある
- [ ] `README.md` のSkill一覧が必要に応じて更新されている
- [ ] `README.md` の説明とSkill本文に矛盾がない
- [ ] `README.md` の使い方とSkill本文に矛盾がない

## 安全性

- [ ] 破壊的変更の扱いが説明されている
- [ ] 依存関係追加の扱いが説明されている
- [ ] 仕様変更の扱いが説明されている
- [ ] テスト不足の扱いが説明されている
- [ ] APIキー、トークン、パスワードなどの秘密情報を出力しない方針がある
- [ ] `scripts/` を使う場合は、目的、危険性、実行条件が説明されている

## 出典・ライセンス

- [ ] 参考元がある場合はライセンス確認済みである
- [ ] コピーではなく、自分の言葉で書き直している
- [ ] コピー・改変がある場合は元ライセンスに従っている
- [ ] 必要な出典が `NOTICE.md` または該当Skill内に書かれている
- [ ] ライセンス不明の内容を取り込んでいない
