# Dogfooding Notes

このドキュメントは、初期Skill群を実際にCodexで使った結果を記録するための手動検証ログです。

実際に検証していない内容を「検証済み」として扱わないため、記入例と実記録を分けて管理します。

## 記録方法

1. [skill-usage-examples.md](skill-usage-examples.md) から依頼例を1つ選ぶ。
2. Codexに依頼し、どのSkillが使われたかを確認する。
3. 期待したSkill、実際に使われたSkill、出力の評価、改善候補を記録する。
4. 改善が必要な場合は、該当Skillの `description`、使う場面、使わない場面、出力形式を見直す。
5. Skill本文を変更した場合は、[skill-review-checklist.md](skill-review-checklist.md) でも確認する。

## 依頼タイプ

- 日本語: 日本語でSkill名や目的を明示した依頼。
- English: 英語のトリガー語を含む依頼。
- 曖昧: Skill名を明示しない、または目的が少し広い依頼。

## 出力の評価

- 良い: 期待したSkillが使われ、出力形式や内容も実用上問題ない。
- 要改善: 期待したSkillは使われたが、description、境界判断、出力形式などに改善余地がある。
- 不一致: 期待したSkillと異なるSkillが使われた、または出力が目的と合わない。

## 検証観点

- [ ] 期待したSkillが使われたか
- [ ] `description` のトリガー語は十分か
- [ ] 似たSkillと誤判定されなかったか
- [ ] 出力形式は使いやすかったか
- [ ] 未確認事項やリスクが明示されたか
- [ ] 日本語で自然に使えたか
- [ ] 英語の依頼でも期待通り使えたか
- [ ] 曖昧な依頼でも適切に境界判断できたか
- [ ] READMEの使い方と矛盾しなかったか
- [ ] AGENTS.mdの方針と矛盾しなかったか
